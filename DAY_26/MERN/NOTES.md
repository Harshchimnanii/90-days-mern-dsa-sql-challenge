# Day 26 – Engineering Thinking (Project Deep Dive)

## Feature Chosen
Expense tracking feature

## Why This Feature Exists
- Helps users monitor spending
- Solves real-world money management problem

## System Flow
Frontend (React)
→ API call
→ Backend (Node / Express)
→ Database (MongoDB)
→ Response back to UI

## State Management
- useState for local UI state
- Data stored in backend for persistence

## Scaling for 10k Users
- Pagination for large lists
- Backend validation
- Indexing database fields
- Caching frequently used data

## Learning
- Interviewers expect system thinking
- Explanation > fancy features
